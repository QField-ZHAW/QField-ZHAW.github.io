2  Projekt aufsetzen

Hinweis

Die nachfolgenden Schritte müssen pro Projekt gemacht werden.

Um die Anleitung etwas verständlicher zu machen, spielen wir ein Beispiel durch. In diesem Beispiel werden verschiedene Vegetationstrukturen erfasst:

  • Wiesen und Weiden (Flächenhafte ausprägung)
  • Hecken (Linienstrukturen)
  • Einzelbäume und Sträucher (Punkte)

2.1 Schritt 1: qgis Projekt erstellen

Als erster Schritt wird mit qgis Desktop ein Projekt erstellt (Project → New). Wir speichern das Projekt mit dem Namen Vegetation-CH an einem sinnvollen Ort auf unserem Computer ab (Project → Save As).

2.2 Schritt 2: Geopackage erstellen

Sämtliche Vektordaten werden in qfield in sogenannten Geopackages erfasst. Geopackages sind alternativen zu Shapefiles und ESRI Geodatabases.

Um ein Geopackage zu erstellen, rechtsklick auf Browser Panel → Project Home und danach New → Geopackage. Beachtet, dass Project Home ist nur sichtbar ist, wenn das qgis Projekt abgespeichert wurde.

Das Geopackage erscheint mit dem Namen New Geopackage.gpkg unter Project Home (siehe Abbildung 2.1). Wir benennen das File in Felderhebung.gpkg um (Rechtsklick → Manage → Rename..).

2.3 Schritt 2: Geopackage layer erstellen

Für jeden Datentyp (Punkte, Linien, Polygone), der Erfasst werden soll, wird ein Geopackage Layer erstellt. Dafür klickt man mit Rechts auf das Geopackage → New Table. Nun muss ein Layer Name (Name), ein Geometrietyp (Geometry Type) sowie ein Koordinatenbezugssystem (CRS) ausgewählt werden (siehe Abbildung 2.2). Für die Schweiz wird das CRS CH1903+ LV95 mit dem EPSG Code 2056 empfohlen.

Für unsere Datentypen verwenden wir die Layer Namen und Geometrietypen wie in Tabelle 2.1 beschreiben. Beachtet, dass die Layernamen weder Umlaute, Sonderzeichen oder Leerzeichen enthalten sollen.

Wir verwenden in unserem Beispiel (Tabelle 2.1) einfache Point / LineString / Polygon. Zu diesen Geometrietypen stehen noch MultiPoint, MultiLineString und MultiPolygon zur Verfügung. Diese werden dann verwendet, wenn sich mehrere Objekte alle Attribute teilen. Mehr zu Attributen im nächsten Unterkapitel.

2.4 Schritt 3: Attribute definieren

Damit wir die Beobachtungen, die wir im Feld machen auch festhalten können, brauchen wir Spalten (sog. Attribute). Diese Attribute (engl: Fields) werden pro Layer separat erstellt. Für jede Spalte wird ein Datentyp ausgewählt, wobei folgende Datentypen am häufigsten sind:

  • Nummerische Werte (mit / ohne Nachkommastellen)
  • Text
  • Datum (mit / ohne Uhrzeit)
  • Bilder

Spalten können entweder direkt während der Erstellung des Layers (siehe Abbildung 2.2) oder auch nachträglich hinzugefügt werden. Für ein nachträgliches Hinzufügen von Spalten muss der Layer dem Projekt hinzugefügt werden (Rechtscklick → Add Layer to Project).

Danach im Layers Panel mit Rechtsklick auf den Layer → Properties → Fields. Nach Klick auf 🖉 Toggle Editing Mode können witere Spalten hinzugefügt werden.

Wir erstellen pro Layer die nachstehenden Spalten. Die Namen der Felder sollten keine Umlaute, Leerschläge oder Sonderzeichen enthalten. Allerdings kann pro Feld ein Alias definiert werden: Dabei handelt es sich um eine alternative Bezeichnung, bei dem diese Regeln nicht gelten.

Detailliertere Informationen zu den Datentypen in Geopackages findet ihr in Tabelle 3.1

2.5 Schritt 4: Attributeingabe festlegen

Um den Input von Attribute zu vereinfachen, können Eingabefelder festgelegt werden. Beispielsweise für solche Vereinfachungen:

  • Eingabe von numerischen Werten via einem Slider (zum Beispiel wenn sich der Wert nur in einem bestimmten Bereich bewegen kann)
  • Eingabe von Ja/Nein mittels Checkbox
  • Eingabe von Text über ein Dropdown Menü (wenn es eine festgelegte beschränkte Liste von Optionen gibt)
  • Eingabe von Datum via einem Kalender

Zudem können Datenqualitätsaspekte festgelegt werden. Beispielsweise:

  • Ein Feld darf nicht leer sein (zwingende Eingabe)
  • Jeder Wert darf nur 1x vorkommen (Eindeutige Werte)
  • Ein Feld wird automatisch ausgefüllt und darf nicht editiert werden (zum Beispiel fid)

Weiter können auch kleinere Erleichterungen eingebaut werden. Beispielsweise, dass der Defaultwert eines Feldes der letzten Eingabe entspricht. Dies kann sinnvoll sein, wenn absehbar ist, dass oft die gleichen Attribute nacheinander eingetragen werden müssen (z.B. Erfassungsdatum.)

Diese Aspekte werden im qgis Projekt festgehalten. Rechtsklick auf den Layer Properties → Attribute Form

In unserem Fiktiven Beispiel können wir folgende Formulareingaben festhalten:

  • Wiesen_Weiden sowie Hecken
    • Datum:
      • Folgende Standarteinstellungen für Datumsfelder sind bereits ausreichend (Widget Type: Date/Time, 🗹 Calender Popup)
      • Um die letzte Eingabe als Defaultwert für neue Objekte zu machen, setzen wir noch 🗹 Reuse last entered value
    • Typ:
      • Hier hätten wir gerne eine Dropdownliste verschiedener Wiesen/Weiden bzw. Heckentypen. Für Wiesen/Weiden: soll es nur zwei mögliche Einträge geben: Wiese sowie Weide
    • Anzahl Arten
    • Bemerkung

  1. Bei Text kann optional die maximale Textlänge angegeben werden. Der Defaultwert 0 bedeutet “unbeschränkt”↩︎

  2. Zahlen ohne Nachkommastellen im Wertebereich ca. -2.1 bis +2.1 Mio. Für grössere Zahlen 64 Bit verwenden↩︎

  3. eine Fliesskommazahl↩︎
