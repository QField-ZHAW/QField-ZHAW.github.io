Wir haben an der ZHAW eine QField Cloud Organization mit Acacemic pricing. Dies beinhaltet unter anderem folgendes (siehe auch hier bzw. hier):

  • Unlimitierte Anzahl von Projekten (private und public)
  • Unlimitierte Anzahl von Nutzern
  • 50 GB Speicherplatz (mehr auf Anfrage / Mehrkosten)
  • Permium Support bei Anfragen
  • Pro features wie Versionierung und Offline PostGIS layers

Unsere Organisation heisst ZHAW, die URL für unsere Organisation lautet:

https://app.qfield.cloud/a/ZHAW/

4.1 Organisationsmitglieder hinzufügen

Wir können Mitglieder zu Organisation via Emailadresse oder Benutzernamen hinzufügen.

Bei Hinzufügen via Email: Falls kein QField-Cloud Account unter der angegebenen Emailadresse registriert ist, wird eine Einladung zur Registrierung an die angegebene Mailadresse verschickt.

  • Einzelne Nutzer können via dem Browserinterface hinzugefügt werden, siehe https://app.qfield.cloud/settings/ZHAW/members.
  • Mehrere Nutzer (z.B. eine ganze Schulklasse oder alle IUNR Mitarbeiter) können via der QField Cloud API hinzugefügt werden, siehe dazu Kapitel 5.2

4.2 Projektmitglieder hinzufügen

Um gemeinsam an einem QField Projekt zu arbeiten, müssen die Projektmitglieder (sogenannte Collaborators) dem Projekt hinzugefügt werden. Um die Berechtigung der einzelnen Mitglieder zu steuern gibt es verschiedene Rollen. Diese sind in Tabelle 4.1 zusammengefasst. Mehr Infos dazu sind hier zu finden.

4.3 Organization Teams

Teams ermöglichen es, auf einfache Weise Berechtigungen für mehrere Benutzer gleichzeitig zu vergeben (z.B. eine Forschungsgruppe oder Klasse von Studenten).

Ein Team besteht aus mehreren Nutzer der Organisation. Wenn einem Team eine Rolle in einem Projekt zugewiesen wird, haben alle Teammitglieder automatisch auch diese Rolle (siehe Kapitel 4.2). Ein Nutzer kann an mehreren Teams beteiligt sein.

  1. Inserting does not include updating and deleting data.↩︎

  2. Editing includes updating and deleting data.↩︎
